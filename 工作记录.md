## 5. 下一步建议

如果要继续往课程项目可交付版本推进，建议按下面顺序补：

1. 确认硬件型号和通信协议
2. 把三类传感器驱动补到接口实现类
3. 确认人脸识别模型或人脸库输入格式
4. 把 `JsonlControlCenterClient` 换成真实 HTTP / ROS 上报
5. 把 `SimpleMotionController` 换成真实底盘控制接口
6. 加入日志模块与参数热更新
7. 增加真实场景测试数据

## 6. 当前默认业务假设

本实现当前采用的默认假设如下：

1. 非法人员 = 限制区域内未授权人员
2. 火灾 = 烟雾 + 温度联合触发
3. 漏电 = 电压/电流异常或漏电流异常
4. 机器人端异常处置只负责“上报 + 离开”
5. 更复杂的人工处置闭环由控制中心负责
